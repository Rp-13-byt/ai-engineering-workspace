# Contributing

1. Create a branch from `main`.
2. Keep changes scoped to one feature or fix.
3. Add or update tests for behavior changes.
4. Run backend and frontend tests before opening a PR.
5. Update documentation for API, schema, deployment, or security changes.

Commit style:

```text
feat: add repository indexing worker
fix: enforce organization access on tasks
docs: explain vector search tradeoffs
```
